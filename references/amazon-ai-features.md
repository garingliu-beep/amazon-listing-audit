# 亚马逊 AI 功能参考

> **⚠️ 重要变更：2026年5月13日，亚马逊关停独立的 Rufus，将其整合进 "Alexa for Shopping"。本文档已更新。**

## 一、75 字符标题政策（2026年7月27日起）

- 所有商品标题 ≤ 75 字符（含空格），超过将被 AI 自动建议更新
- 标题职责从"承载所有关键词"转变为"定义产品身份"
- 关键词迁移到 Product Highlights、后台属性和搜索词
- **品牌名计入 75 字符限制**（已确认）
- **75 字符限制适用于父 ASIN 和子 ASIN**（已确认）
- **套装也适用 75 字符限制**（已确认）
- 媒体类（书/音乐/视频/DVD）豁免
- 时尚/鞋靴/珠宝父 ASIN：美加 125 字符，日本 130 字符

**来源：** Seller Central 公告（2026年6月10日）+ Seller Central 论坛讨论帖

## 二、Product Highlights（2026年7月27日起新增）

- 125 字符，显示在搜索结果页和详情页标题下方
- **可被搜索、参与搜索索引**（亚马逊官方已确认）
- 逗号分隔短语形式，不使用完整句子
- 职责：描述核心属性（材质/功能/场景/人群/规格）
- **与五点描述是独立模块**（已确认）
- **目前为选填，但受内容监控**（已确认）
- 不重复标题中已有的关键词（浪费索引空间）

**来源：** Seller Central 论坛回复

**Listing 模块职责分工：**

**⚠️ 前台显示机制：** 亚马逊前台将 Title 和 Product Highlights **拼接显示**，系统自动在中间插入 `|` 分隔符。买家实际看到的是：
```
{Title} | {Highlights}
```
这是一个连续文本，不是两个独立区域。因此：
- 75+125 是搜索索引层面的字符分配
- 视觉展示是拼接后的整体，需考虑可读性
- 关键词不得在两个字段间重复（买家看到重复词会困惑）

| 模块 | 职责 | 限制 | 搜索索引 | Alexa 读取 |
|------|------|------|---------|-----------|
| Title | 定义产品身份 | ≤75 字符 | ✅ 最高权重 | ✅ |
| Product Highlights | 描述核心属性 | ≤125 字符 | ✅ 已确认 | ✅ |
| Bullet Points | 说服和转化 | 5条×500-1000字符（推荐150-255） | ✅ 权重因品类而异 | ✅ |
| A+ Content | 深度展示+品牌故事 | 模块化 | ❌ 文本不索引（alt text 可能） | ✅ |
| Search Terms | 扩展索引覆盖 | 250字节 | ✅ | ❌ 不可见 |
| 后台属性 | 结构化数据 | 因品类而异 | ✅ CDQ 30%权重 | ✅ |
| Q&A | 买家问答 | 不限 | ✅ | ✅ **核心数据源** |
| Review | 用户评价 | 不限 | ✅ 间接 | ✅ **核心数据源** |
| 图片 | 产品展示 | 9张 | ❌ | ✅ **OCR 扫描文字** |

## 三、三大系统架构（2026年5月后）

亚马逊搜索与发现由三个分层协作的系统驱动，**不是三选一，而是三层叠加**：

```
┌─────────────────────────────────────────────┐
│         用户搜索 / 提问 / 对话                │
│   （关键词搜索 或 自然语言提问 Alexa）         │
└──────────────────┬──────────────────────────┘
                   │
      ┌────────────▼────────────┐
      │  Alexa for Shopping     │ ← 交互层（原 Rufus + Alexa+）
      │  对话式购物 / AI 总结    │   用户直接感知
      │  代下单 / 跨站比价       │   2026年5月13日上线
      └────────────┬────────────┘
                   │
      ┌────────────▼────────────┐
      │        COSMO            │ ← 个性化层
      │  语义知识图谱 / 千人千面  │   用户画像匹配
      │  意图理解 / 场景关联      │   上下文搜索
      └────────────┬────────────┘
                   │
      ┌────────────▼────────────┐
      │         A10             │ ← 排名引擎
      │  关键词匹配 / 转化率     │   搜索结果排序
      │  卖家权威 / 销售速度     │   索引决定
      └─────────────────────────┘
```

### 3.1 A10 算法（底层排名引擎）

- 决定搜索结果中哪些产品出现、排第几
- **转化率是最重要的单一变量**（12% CVR 的 Listing 排名高于 8%）
- A10 降低 PPC 销售权重，提升有机销售和外部流量权重
- 点击率（CTR）直接 feed 进 A10 质量信号
- Review 速度和近期度 > 总评论数
- 奖励：关键词相关性、转化率、CTR、卖家权威、外部流量

**A10 排名因素权重（NovaData 200+ Listing A/B 测试，数据时间待确认）：**
- 销售速度：35-40%
- 相关性与关键词：25-30%
- 客户满意度：20-25%

### 3.2 COSMO（中间个性化层）

- 从 Keyword Search 升级为 Intent Search
- 理解用户搜索背后的深层意图和生活场景
- 分析：购买历史、浏览习惯、用户偏好、地理位置、季节、天气
- **全球统一的固定排名已不复存在** — 同一产品在不同用户/时间/地点的排名可能不同
- 在 10% 美国搜索流量测试中（Amazon 内部数据，时间待确认）：购买量 +0.7%，购物者参与度 +8%
- 后台属性是 AI 构建知识图谱的数据基础

### 3.3 Alexa for Shopping（顶层交互层）

**2026年5月13日上线，整合了 Rufus + Alexa+ 的全部能力。**

**核心能力：**

| 能力 | 说明 | 对卖家的影响 |
|------|------|-------------|
| 搜索栏直接对话 | 买家在搜索框直接提问，AI 回答 | Listing 要能回答具体问题 |
| AI 自动总结 Listing | 搜索结果顶部和详情页展示 AI 生成总览 | 信息必须自洽、可验证 |
| AI 代下单 | "帮我买下来" / 价格触达阈值自动购买 | 定价策略透明度上升 |
| 跨设备记忆 | Echo/App/网站之间偏好数据双向流动 | 个性化匹配概率成新维度 |
| 跨站比价 | Shop Direct 功能，全网比价 | 竞争从站内扩展到全网 |
| 价格历史展示 | 任意商品过去一年价格历史 | 避免频繁无规律调价 |
| 免费开放 | 所有 Amazon 用户可用，无需 Prime/Echo | 用户基数快速膨胀 |

**Alexa 读取 Listing 的方式：**
- 深度读取：标题、Bullet、描述、A+、图片（OCR）、Review、Q&A、产品参数表
- **提取+总结**，不是逐字阅读
- 信息混乱/矛盾/空泛 → AI 总结可能偏离卖家意图
- Review 中高频正面/负面词 → 直接影响 AI 推荐

**数据：**
- 原 Rufus 用户 2.5 亿+（已整合进 Alexa）
- 原 Rufus 增量销售 $100 亿/年
- 原 Rufus 用户转化率比非用户高 60%
- Alexa for Shopping 免费开放后用户基数持续扩大

**Alexa 对流量分配的影响（2026年5月后）：**
- **流量分发**：从"关键词排名"到"AI 语义匹配"
- **流量分配**：从"普惠分发"到"头部集中 + 细分倾斜"
- **转化逻辑**：从"页面说服"到"AI 背书"
- **QA 优先级**：AI 抓取 QA 的优先级**高于标题**
- **价格稳定性**：频繁调价被 AI 判定为"低信任度"
- **虚假评价识别**：Alexa 具备强大的虚假评价识别能力

**亚马逊新增广告形式 — Prompts 对话广告：**
- 从"关键词触发"转向"用户意图触发"
- 投放高意向对话场景
- 优先布局场景化长尾词，匹配 AI 语义推荐逻辑

**来源：** Amazon 官方公告（2026年5月13日）+ 雨果网/雪球/AMZ123/百家号报道（2026年5月，转载解读）

### 新流量公式
```
流量 = 关键词排名(A10) × AI推荐概率(Alexa) × 个性化匹配概率(COSMO)
```

## 四、AI 生成 Listing 内容

亚马逊提供 AI 辅助生成标题/Bullet/描述/A+/Highlights 的工具。
审计时检查：是否使用了 AI 工具？是否人工优化？是否有"幻觉"内容？

## 五、AI Review 分析

Alexa for Shopping 从 Review 中提取信息来回答买家问题。
审计时检查：
- 正面 Highlights 是否与 Listing 卖点一致
- 负面 Highlights 是否在 Listing 中已有回应
- **Q&A 区域是否有未回答的问题**（未回答 = 错失 Alexa 推荐机会）

## 六、视频在 AI 搜索中的作用

- 亚马逊多模态 AI 可分析视频内容
- 视频中的产品信息可能被 AI 提取用于推荐
- 有视频的 Listing 通常有更高点击率

**视频分析方法：** 使用 `mimo-omni` skill（`bash mimo_api.sh video`）

## 七、Alexa for Shopping 购物入口重塑

**购物路径变化：**
- 过去：搜索关键词 → 浏览 → 对比 → 下单
- 现在：向 AI 描述需求 → AI 理解意图 → 推荐商品 → 代下单

**6 条新规则（2026年5月13日起）：**
1. 搜索栏支持直接提问对话
2. AI 自动总结 Listing
3. AI 代下单 + 价格历史展示
4. 跨设备记忆用户画像
5. AI 可跨站比价
6. 购物入口免费开放

**Alexa 模拟测试问题：**
1. "这个产品适合什么人用？"
2. "什么场景下使用？"
3. "核心材质/技术是什么？"
4. "和竞品有什么区别？"
5. "安装难度如何？"
6. "Review 里最常见的问题是什么？"
7. "如果降到 XX 钱帮我加入购物车"
8. "帮我在全网找更便宜的"
