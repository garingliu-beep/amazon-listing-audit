# 品类配置：通用模板

> 所有品类配置文件继承此模板。品类配置只需定义**差异部分**，共性部分从本模板继承。

---

## 一、品类信息（每个品类必须填写）

```yaml
category_name: "{品类名称}"
category_name_en: "{英文品类名}"
amazon_paths:
  - "{类目路径1}"
  - "{类目路径2}"
identification_keywords:
  - "{识别关键词1}"
  - "{识别关键词2}"
```

---

## 二、通用属性（所有品类共享）

以下属性适用于所有品类，品类配置可在此基础上**新增**专属属性。

| 属性 | 重要性 | 说明 |
|------|--------|------|
| Brand | 必填 | 品牌名 |
| Material | 必填 | 产品主体材质 |
| Color | 必填 | 颜色 |
| Finish Type | 必填 | 表面处理工艺 |
| Style | 选填 | 风格（Modern/Traditional/Transitional） |
| Item Weight | 选填 | 产品重量 |
| Product Dimensions | 选填 | 产品尺寸 |
| Included Components | 选填 | 包含组件 |
| Warranty | 选填 | 保修信息 |
| Country of Origin | 选填 | 产地 |

---

## 三、通用 Review 语义聚类主题（所有品类共享）

以下 6 个主题适用于所有品类，品类配置可**替换或扩展**。

| # | 主题 | 关注点 |
|---|------|--------|
| 1 | 安装体验 | 安装难度、配件齐全度、说明书清晰度 |
| 2 | 质量耐久 | 材质手感、使用寿命、是否生锈/褪色 |
| 3 | 外观设计 | 颜值、与装修风格搭配、质感 |
| 4 | 功能体验 | 核心功能是否好用、操作顺畅度 |
| 5 | 材质感受 | 重量感、手感、是否有异味 |
| 6 | 售后服务 | 退换货、客服响应、配件补发 |

## 四、通用合规要求（所有品类共享）

### 4.1 标题合规
- [ ] 字符数 ≤ 75（含空格）
- [ ] 首字母大写（非全大写）
- [ ] 不使用特殊符号
- [ ] 不包含促销信息/主观评价词/价格/卖家信息
- [ ] 品牌名与注册品牌一致

### 4.2 图片合规
- [ ] 主图纯白背景（RGB 255,255,255）
- [ ] 产品占比 ≥ 85%
- [ ] 最长边 ≥ 1000px（推荐 2000px+）
- [ ] 格式：JPEG/PNG/TIFF/GIF
- [ ] 颜色模式：sRGB
- [ ] 无文字/标志/水印/边框

### 4.3 五点描述合规
- [ ] 不包含卖家信息/促销信息
- [ ] 不使用 HTML 标签
- [ ] 认证声明真实可验证

### 4.4 搜索词合规
- [ ] 250 字节内
- [ ] 不包含竞品品牌名/ASIN/商标词
- [ ] 空格分隔，不用逗号
- [ ] 不重复标题中已有的词

### 4.5 知识产权
- [ ] 图片原创或有授权
- [ ] 不使用竞品品牌名或商标
- [ ] 品牌名不与已有品牌过于相似

## 五、通用站点合规要求

| 站点 | 要求 |
|------|------|
| 美国站 | Prop 65 警告、FTC 合规 |
| 加拿大站 | 双语（英法） |
| 欧洲站 | CE 标记、REACH 合规 |
| 英国站 | UKCA 标记 |
| 日本站 | PSE 标记（电气类） |
| 澳大利亚站 | WELS 评级（水相关） |

## 六、通用评分权重（14维度）

| # | 维度 | 权重 |
|---|------|------|
| 1 | 标题优化 | 11% |
| 2 | Product Highlights | 7% |
| 3 | 五点描述 | 9% |
| 4 | 产品描述/A+ | 5% |
| 5 | 后台搜索词 | 5% |
| 6 | 实体SEO与关键词 | 9% |
| 7 | 后台属性完整性 | 9% |
| 8 | 图片检查 | 12% |
| 9 | 定价与促销 | 5% |
| 10 | Review分析 | 7% |
| 11 | 竞品对比 | 3% |
| 12 | 类目选择 | 3% |
| 13 | 合规性检查 | 5% |
| 14 | 表现信号 | 10% |

> 品类配置可调整权重（如某品类合规特别重要，可上调维度13权重），但必须确保总和 = 100%。

## 七、品类配置文件结构规范

每个品类配置文件必须包含以下章节：

```markdown
# 品类配置：{品类名}

## 继承自
_template.md

## 1. 品类信息
（YAML 格式，含类目路径和识别关键词）

## 2. 专属属性
（只列本品类新增的属性，通用属性从模板继承）

## 3. 专属 Review 主题
（只列本品类特有的主题，通用主题从模板继承）

## 4. 专属合规要求
（只列本品类特有的合规项，通用合规从模板继承）

## 5. 专属关键词库
（引用独立关键词文件，或直接列出核心词）

## 6. 评分权重调整
（如有调整则列出，无调整则写"沿用通用权重"）

## 7. 竞品选择补充条件
（品类特有的竞品筛选条件）
```
